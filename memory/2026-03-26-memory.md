## Learnings
- `react-hooks/set-state-in-effect` ESLint rule in Next.js 16 (React 19 compiler rules) blocks `useEffect(() => setState(...))` even for mount detection — use `useSyncExternalStore(emptySubscribe, () => true, () => false)` instead
- Tailwind v4 dark variant `@custom-variant dark (&:is(.dark *))` only targets descendants of .dark, not the .dark element itself — use `(&:where(.dark, .dark *))` for full coverage
- `npx create-next-app@latest` installs whatever is latest (got Next.js 16 instead of planned 15) — pin versions in plan if exact version matters
